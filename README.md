# Lapushka 5

## Назначение проекта
Lapushka 5 — модульная система для загрузки, синхронизации, распознавания, классификации и сопоставления документов с последующим формированием проектных наборов и выявлением нехватки материалов.

## Архитектурный принцип
Система состоит из автономных модулей. Каждый модуль имеет:
- свою папку;
- свой `main.py`;
- свой UI на отдельном порту;
- свою БД;
- свой `README.md`;
- свой `AGENTS.md`.

## Структура проекта
- `modules/` — основные процессники
- `reviewers/` — модули проверки
- `orchestrator/` — координация и маршрутизация задач
- `shell/` — единый фронтовый слой
- `shared/` — общие соглашения и контракты
- `.env.example` — пример переменных окружения
- `AGENTS.md` — общая политика для всех агентов

## Основные модули
### process-модули
- `ingest_registry` — импорт, синхронизация и реестр файлов
- `doc_classifier` — распознавание и классификация документов
- `project_builder` — формирование проектов, связей и нехватки

### reviewer-модули
- `backend_reviewer` — проверка бэкенд-процессов
- `frontend_reviewer` — легкая браузерная проверка UI
- `db_reviewer` — проверка БД
- `prompt_reviewer` — ручная проверка промптов по прямой команде пользователя

### системные модули
- `orchestrator` — управление маршрутизацией задач и статусами
- `shell` — общий стиль и единая точка входа

## Принцип задач
- Источник правды — SQL-реестр задач.
- Google Sheets — только дашборд наблюдения.
- Записи в SQL-реестр не изменяются и не удаляются, только добавляются новые.

## Стадия
На текущем этапе это стартовый каркас проекта.

## Запуск модулей
Для стабильного локального запуска process-модулей используйте единый контроллер:
- `./modulectl start ingest_registry`
- `./modulectl start doc_classifier`
- `./modulectl start project_builder`
- `./modulectl status <module_name>`
- `./modulectl health <module_name>`
- `./modulectl stop <module_name>`

Контроллер:
- использует фиксированные порты из `.env`;
- хранит pid-файлы и логи в `.runtime/`;
- не запускает второй процесс поверх уже работающего;
- помогает агентам запускать модули одинаково.
