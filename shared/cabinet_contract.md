# Cabinet Contract

## Назначение
`cabinet` — минимальная общая сущность, которая задает системный контекст для shell, task workflow и process-модулей.

## Минимальная модель
- `id` — стабильный идентификатор кабинета, используется в URL, API и БД;
- `slug` — человекочитаемый идентификатор для display и ссылок;
- `title` — отображаемое название кабинета;
- `status` — состояние кабинета (`active`, `archived`);
- `created_at` — время создания.

## Базовые правила
- Любой пользовательский сценарий process-модуля выполняется в контексте выбранного `cabinet_id`.
- `cabinet_id` должен явно передаваться через shell-маршрут, task workflow и модульные API.
- Shell хранит кабинет только как навигационный контекст и не становится источником бизнес-данных кабинета.
- До полной миграции legacy-режим допускает системный кабинет `default`.

## Первый этап внедрения
- shell использует маршруты вида `/cabinet/<cabinet_id>/...`;
- task workflow хранит `cabinet_id` в `tasks`, `task_handoffs`, `task_reviews`, `task_events`;
- если `cabinet_id` не передан явно, используется `default`.
