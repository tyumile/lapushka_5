# UI Contract

## Назначение
Этот документ задает единые базовые принципы интерфейса для всех модулей Lapushka 5.

## Общие требования
- UI должен быть простым, рабочим и легким.
- UI не должен быть декоративным.
- На странице должны быть только полезные элементы.
- Важные действия должны быть видны сразу.
- Результат действия должен быть понятен без чтения кода.

## Общий стиль
- чистая структура страницы;
- явные заголовки;
- компактные формы;
- простые таблицы;
- понятные кнопки;
- видимые статусы;
- умеренные отступы;
- без тяжелых эффектов и анимаций.

## Поведение
- страницы не должны падать при пустых данных;
- ошибки должны отображаться понятно;
- длительные действия должны иметь статус;
- действия пользователя должны давать видимый результат.

## Ограничения
- не перегружать CPU и RAM сервера тяжелыми фронтовыми эффектами;
- не использовать тяжелые UI-паттерны без необходимости;
- не строить сложные клиентские состояния без прямой пользы.
