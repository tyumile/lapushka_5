# orchestrator

## Назначение
Модуль orchestrator управляет маршрутизацией задач, статусами работы и handoff между process-модулями и reviewer-модулями.

## Что делает
- координирует поток задач;
- отслеживает этапы выполнения;
- направляет задачи в review;
- фиксирует связи между этапами;
- опирается на SQL-реестр задач как источник правды.

## Что не делает
- не реализует бизнес-логику process-модулей;
- не заменяет shell;
- не хранит у себя чужие внутренние данные без необходимости.

## Структура папки
- `main.py` — точка входа orchestrator
- `AGENTS.md` — локальная политика
- `README.md` — описание роли и структуры
- `app/` — логика координации
- `tests/` — локальные тесты
- `logs/` — локальные логи

## Принцип работы
- SQL-реестр задач — источник правды.
- Google Sheets — только дашборд.
- Любое действие фиксируется новой записью.
- Старые записи не изменяются.
