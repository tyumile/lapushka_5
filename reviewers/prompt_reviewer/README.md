# prompt_reviewer

## Назначение
`prompt_reviewer` — локальный reviewer-модуль для ручной проверки промптов.

Модуль используется только по прямой команде пользователя и работает только с тем промптом, который пользователь явно указал в текущем запросе или передал через stdin.

## Явные ограничения
- модуль не подключается к общей автоматической очереди задач;
- модуль не инициирует проверку сам;
- модуль не анализирует посторонние промпты без прямой команды пользователя;
- модуль не меняет чужой код и не вмешивается в работу других модулей.

## Что делает модуль
- принимает конкретный указанный промпт;
- делает базовый ручной review по ясности, ограничениям, формату и рискам;
- выдает рекомендации по улучшению формулировки.

## Структура папки
- `main.py` — минимальная точка входа для ручного запуска reviewer-модуля
- `app/__init__.py` — пакет модуля
- `app/reviewer.py` — минимальная логика анализа конкретного промпта и форматирования результата
- `AGENTS.md` — локальная политика reviewer-модуля
- `README.md` — описание роли, ограничений и структуры

## Запуск
Прямой запуск с текстом промпта:

```bash
python main.py "Сформируй краткий ответ и обязательно дай 3 пункта."
```

Запуск через stdin:

```bash
echo "Опиши архитектуру сервиса и форматируй ответ списком." | python main.py
```

Если промпт не передан, модуль не начинает никакой автоматический поиск и выводит подсказку по запуску.

## Порт и UI
У модуля нет отдельного UI и нет выделенного порта. Это минимальный reviewer-инструмент для ручного использования внутри папки `prompt_reviewer`.

## База данных
Собственная БД не используется.

## Основные сценарии работы
1. Пользователь явно передает конкретный промпт.
2. Модуль анализирует только этот промпт.
3. Модуль возвращает краткие замечания и рекомендации.
