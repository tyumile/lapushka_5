# db_reviewer

## Назначение
Проверка БД модулей-процессников на чистоту, целостность и понятность структуры.

## Что проверяет
- схему таблиц и понятность имен;
- наличие дублей и критичных пустых полей;
- соответствие структуры роли модуля.

## Что не делает
- не меняет чужой код напрямую;
- не закрывает задачи вместо процессников;
- не становится бизнес-модулем.

## Структура папки
- `main.py` — точка входа reviewer-модуля
- `AGENTS.md` — локальная политика reviewer-а
- `README.md` — описание роли и структуры
- `app/` — логика проверки
- `tests/` — локальные тесты
- `logs/` — локальные логи
- `prompts/` — шаблоны текстов и рекомендаций, если нужны

## Запуск
Команда запуска:
- `python main.py`

## Принцип работы
Reviewer читает завершенные изменения или указанный объект проверки, формирует рекомендации и фиксирует результат новой записью в SQL-реестре задач.
