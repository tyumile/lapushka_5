# db_reviewer

## Назначение
`db_reviewer` готовит и оформляет review БД модулей-процессников. Модуль не правит чужие данные и не запускает миграции: его задача в том, чтобы проверить структуру, зафиксировать замечания и передать понятный результат через реестр задач.

## Что проверяется в БД
- понятность таблиц и колонок;
- логичность хранения статусов, идентификаторов и связей;
- дубли в ключевых сущностях;
- пустые критичные поля;
- битые ссылки и очевидные нарушения целостности;
- соответствие схемы назначению конкретного модуля.

## Какие замечания считаются важными
- `broken_foreign_keys` — ссылки ведут на несуществующие записи;
- `duplicate_rows_in_key_entities` — дубли ломают уникальность сущностей;
- `empty_critical_fields` — отсутствуют значения в колонках, без которых запись теряет смысл;
- `unclear_table_or_column_names` — схема стала непонятной для поддержки;
- `status_or_id_storage_conflicts` — статусы или идентификаторы хранятся непоследовательно и повышают риск ошибок.

## Ограничения reviewer-модуля
- не изменяет чужие БД;
- не мигрирует схемы самовольно;
- не исправляет данные вручную;
- фиксирует только замечания, рекомендации и итог review.

## Как оформляются результаты проверки
Результат review оформляется новой записью в `task_reviews` через команду проекта `/home/dev/lapushka_5/task add ... --action-type review --sync`.
Команда `task` для reviewer должна начинаться с `/home/dev/lapushka_5/task mine --agent db_reviewer`, чтобы брать актуальные reviewer-handoff из SQL workflow-базы.

Минимум для результата:
- `summary` — короткий итог проверки;
- `status` — обычно `reviewed`, либо `blocked`/`failed`;
- `artifacts` — модуль, таблицы, замечания и связанные файлы одной строкой.

Шаблон текста результата лежит в `prompts/review_result_template.md`.

## Структура папки
- `main.py` — минимальная точка входа, печатает описание роли модуля;
- `app/__init__.py` — пакет reviewer-модуля;
- `app/reviewer.py` — базовое описание области проверки и список важных замечаний;
- `prompts/review_result_template.md` — шаблон оформления результата review;
- `tests/` — место для локальных тестов reviewer-модуля;
- `logs/` — каталог для локальных логов;
- `AGENTS.md` — локальная политика reviewer-а;
- `README.md` — описание роли, ограничений и структуры.

## Запуск
Минимальный запуск:

```bash
python main.py
```

## База данных и сценарий работы
Собственной БД внутри `db_reviewer` сейчас нет. Итоги проверки пишутся в общую SQL workflow-базу проекта в таблицы `task_reviews` и `task_events`, а сам reviewer используется как модуль проверки и оформления замечаний.
