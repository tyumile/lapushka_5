# frontend_reviewer

## Назначение
`frontend_reviewer` хранит минимальный каркас для легкой браузерной smoke-проверки UI модулей в headless-режиме.

Модуль не выполняет тяжелые автотесты, не записывает видео и не подменяет process-модули. Его задача: быстро открыть страницу, проверить базовую интерактивность и зафиксировать результат review.

## Что именно проверяется
- страница модуля открывается без явного падения;
- видны ключевые элементы интерфейса;
- базовые кнопки и формы не ломают страницу;
- результат действия отображается в UI;
- ошибки видны пользователю и не скрываются пустым экраном;
- все пользовательские тексты в UI даны на русском языке;
- нет очевидных `404`, `500`, зависаний и пустых состояний без объяснения.

## Базовые проверки
Базовыми считаются только короткие smoke-check сценарии:
- открыть целевую страницу;
- дождаться рендера ключевого блока;
- выполнить 1-2 простых действия;
- убедиться, что интерфейс остался рабочим;
- зафиксировать замечания или подтверждение в реестре задач.

## Как не перегружать сервер
- использовать только headless-режим;
- проверять один основной сценарий за проход;
- не запускать длительные циклы и многократные повторы;
- не загружать большие файлы без необходимости;
- для локальных upload-проверок использовать только маленькие тестовые файлы;
- останавливать проверку после первого понятного результата, а не расширять сценарий.

## Структура папки
- `AGENTS.md` — локальная политика reviewer-модуля.
- `README.md` — описание назначения, ограничений и структуры.
- `main.py` — минимальная точка входа, печатает текущую smoke-check политику.
- `app/__init__.py` — экспорт минимального API reviewer-каркаса.
- `app/reviewer.py` — описание легкой browser-smoke политики и чеклиста.
- `prompts/browser_smoke_checklist.md` — краткий шаблон действий для ручной проверки.
- `tests/fixtures/README.md` — правила для маленьких локальных файлов, если нужна upload-проверка.
- `logs/.gitkeep` — сохранение директории под легкие локальные логи.

## Запуск
Минимальный запуск:

```bash
python main.py
```

Команда выводит текущую политику reviewer-модуля и границы допустимой smoke-проверки.

## Локальные тестовые файлы
Папка `tests/fixtures/` подготовлена для редких локальных upload-проверок. В нее следует класть только маленькие и безопасные тестовые файлы, которые не создают заметной нагрузки на UI и сервер.

## Результат работы
Результат reviewer-модуля — новая запись review в `task_reviews` с итогом легкой браузерной проверки: подтверждение, замечание или блокер.
Команда `task` для reviewer должна начинаться с `../../task mine --agent frontend_reviewer`, чтобы брать только свои непроверенные handoff из SQL workflow-базы.
