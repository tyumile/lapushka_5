Lapushka 5

Единый shell-контейнер для трех рабочих зон

Верхняя навигация остается в shell на порту `8000`, а локальные UI модулей подгружаются ниже в отдельную область без переноса их логики в общий слой.

Место shell в системе

front layer
  • служит стартовой страницей общего входа;
  • показывает верхний уровень системы без бизнес-логики модулей;
  • ведет пользователя к локальным UI, а не заменяет их.

Рабочая область

sources

Shell встраивает UI выбранного модуля ниже и сохраняет верхнюю навигацию на месте.

порт 8001 Открыть модуль отдельно

Принципы навигации

  • каждый модуль открывается по своей прямой ссылке;
  • shell хранит только краткое описание и точку входа;
  • подключение новых модулей делается минимальным изменением shell.

Границы shell

  • не переносит бизнес-логику в общий слой;
  • не переписывает локальные страницы модулей;
  • не берет на себя роль оркестратора или центра управления.

Служебные модули без UI

service layer
orchestrator Координация задач и handoff, но не общий UI и не бизнес-логика.
frontend_reviewer Проверка качества интерфейсов, без собственного пользовательского UI.
backend_reviewer Review серверной логики и рисков изменений.
db_reviewer Проверка структуры БД и схем изменений.
prompt_reviewer Ручной reviewer-модуль без отдельного HTTP-интерфейса.

Границы общего фронта

shell only
  • shell не переносит предметную логику модулей в общий слой;
  • shell не встраивает локальные страницы модулей в свой HTML;
  • shell не управляет их запуском и не заменяет orchestrator;
  • shell дает единое ощущение системы только на уровне входа и навигации.
