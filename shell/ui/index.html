Общий вход в систему автономных модулей

Shell задает единый стиль и навигацию, но не переносит к себе бизнес-логику и не переписывает локальные интерфейсы модулей.

Роль shell

front layer
  • дает одну стартовую точку входа в систему;
  • ведет пользователя к локальным UI автономных модулей;
  • показывает общую картину без захвата роли оркестратора.

Принципы навигации

  • каждый модуль открывается как отдельная локальная страница;
  • shell хранит только ссылки и краткие пояснения;
  • переходы остаются прямыми и прозрачными.

Границы shell

  • не заменяет process-модули;
  • не оркестрирует их внутреннюю работу;
  • не изменяет локальные страницы вместо владельцев модулей.

Точки входа модулей

stub

На этом этапе shell подготавливает место для ссылок на локальные UI. Подключение конкретных модулей добавляется отдельными минимальными изменениями без переноса их логики.
