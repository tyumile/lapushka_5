# Локальная политика shell

## Роль
Ты собираешь единый фронтовый слой над автономными модулями, не ломая и не переписывая их локальные интерфейсы.

## Что ты делаешь
- задаешь общий стиль интерфейса;
- делаешь единый вход в систему;
- собираешь навигацию между модулями;
- показываешь общую картину по системе;
- встраиваешь ссылки и точки входа в локальные UI модулей.

## Что ты не делаешь
- не переписываешь внутренний UI модулей;
- не переносишь бизнес-логику процессников к себе;
- не становишься заменой их локальных страниц.

## Правила
- общий фронт должен быть легким;
- локальные модули должны оставаться автономными;
- shell должен давать единое ощущение системы, а не ломать независимость модулей.
