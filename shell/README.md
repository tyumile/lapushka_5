# shell

## Назначение
Shell — отдельный модуль общего фронтового слоя системы. Он собирает единое ощущение интерфейса и одну стартовую точку входа, но не подменяет собой автономные модули.

## Место shell в системе
- shell стоит над локальными UI как легкий входной слой;
- shell связывает пользователя с автономными модулями через навигацию и ссылки;
- shell не переносит к себе бизнес-логику, базы данных или внутренние сценарии process-модулей;
- shell не берет на себя роль оркестратора.

## Общий стиль интерфейса
- спокойный единый визуальный язык для всех входных страниц;
- легкий интерфейс без тяжелых зависимостей и перегруженных элементов;
- акцент на полезных действиях: куда перейти, что это за модуль, зачем он нужен;
- shell задает стиль входа, а не переписывает локальные страницы модулей.

## Принципы навигации
- shell показывает только верхний уровень системы;
- каждая карточка или ссылка ведет в локальный UI конкретного модуля;
- переходы должны быть прямыми и понятными без скрытой логики;
- подключение модулей делается отдельными минимальными изменениями, без переписывания их интерфейсов внутри shell.

## Что shell не делает
- не переписывает локальные UI модулей;
- не переносит бизнес-логику модулей в свой код;
- не становится единым центром управления модулями;
- не заменяет страницы, за которые отвечают владельцы модулей.

## Структура папки
- `main.py` — точка входа, поднимает простой HTTP-сервер для общего UI;
- `AGENTS.md` — локальная политика агента для shell;
- `README.md` — роль shell, границы, структура и запуск;
- `ui/index.html` — стартовая страница общего входа в систему;
- `ui/styles.css` — базовый стиль shell-интерфейса;
- `tests/` — место для тестов shell-модуля;
- `logs/` — директория для логов shell.

## Запуск
- Порт по умолчанию: `8030`
- Хост по умолчанию: `127.0.0.1`
- Точка входа: `main.py`
- Локальный запуск: `python3 main.py`
- Переопределение адреса: через `SHELL_HOST` и `SHELL_PORT`

## Основной сценарий работы
1. Пользователь открывает shell как общий вход в систему.
2. Shell показывает роль системы, границы модуля и общие точки входа.
3. Пользователь переходит в локальный UI нужного автономного модуля.
4. Внутренняя работа модуля остается внутри самого модуля.
