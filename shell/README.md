# shell

## Назначение
Shell — отдельный модуль общего фронтового слоя системы. Он дает единый вход и общий визуальный язык для верхнего уровня интерфейса, не подменяя собой автономные модули.

## Место shell в системе
- shell стоит перед локальными UI как легкий входной слой;
- shell собирает навигацию и краткое описание точек входа;
- shell не хранит бизнес-логику модулей и не управляет их внутренними сценариями;
- shell не становится оркестратором системы.

## Общий стиль интерфейса
- простой общий экран входа без тяжелых зависимостей;
- единая типографика, палитра и карточная подача верхнего уровня;
- акцент только на полезной информации: куда перейти и за что отвечает модуль;
- локальные UI модулей не переписываются shell-модулем и остаются автономными.

## Принципы навигации
- shell показывает только верхний уровень системы;
- каждая верхняя кнопка открывает shell-маршрут вида `/cabinet/<cabinet_id>/<module_name>` на `8000`, а shell проксирует HTML выбранного модуля внутрь общего layout;
- shell хранит кабинет только как навигационный контекст, а не как источник бизнес-данных;
- на входной странице shell можно создать новый кабинет по названию или выбрать существующий;
- подключение нового модуля делается минимальным изменением в shell через новый route и порт модуля.

## Границы shell
- не переносит бизнес-логику модулей в общий слой;
- не переписывает локальные страницы модулей;
- не подменяет локальные интерфейсы общим UI;
- не берет на себя запуск, координацию или оркестрацию process-модулей.

## Структура папки
- `main.py` — точка входа, поднимает HTTP-сервер shell и проксирует HTML модулей в общий layout;
- `main.py` также обрабатывает создание кабинета, выбор кабинета и cabinet-aware маршруты;
- `AGENTS.md` — локальная политика агента для shell;
- `README.md` — описание роли, границ, структуры и запуска;
- `data/cabinets.json` — локальный навигационный реестр созданных кабинетов shell;
- `ui/index.html` — стартовая страница общего входа;
- `ui/sources.html` — shell-страница раздела `Источники`;
- `ui/documents.html` — shell-страница раздела `Рабочая документация`;
- `ui/project.html` — shell-страница раздела `Проект`;
- `ui/styles.css` — базовый визуальный стиль shell;
- `ui/app.js` — легкая клиентская логика для построения ссылок на UI-модули по текущему хосту;
- `tests/test_cabinets.py` — минимальные проверки создания кабинетов и объединения env/persisted списка;
- `tests/.gitkeep` — заготовка под тесты shell-модуля;
- `logs/.gitkeep` — директория для логов и служебных файлов shell.

## Запуск
- Порт по умолчанию: `8030`
- Хост по умолчанию: `127.0.0.1`
- Точка входа: `main.py`
- Локальный запуск: `python3 main.py`
- Переопределение адреса: через `SHELL_HOST` и `SHELL_PORT`

## Основной сценарий работы
1. Пользователь открывает shell как общий вход в систему.
2. На входной странице пользователь создает новый кабинет по названию или выбирает существующий.
3. Shell открывает маршрут вида `/cabinet/<cabinet_id>/sources`, `/cabinet/<cabinet_id>/documents` или `/cabinet/<cabinet_id>/project`.
4. Контент выбранного модуля подгружается shell-сервером и выводится в общий layout страницы.
5. Shell сохраняет только навигационный контекст кабинета, а внутренняя логика модуля остается внутри самого модуля.
