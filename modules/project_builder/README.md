# project_builder

## Назначение
Формирование проектов, сопоставление документов и определение нехватки материалов.

## Границы ответственности
Что модуль делает:
- группирует документы в проекты;
- строит связи между документами;
- показывает полноту набора;
- определяет нехватку.

Что модуль НЕ делает:
- не импортирует файлы;
- не делает OCR;
- не выполняет первичную классификацию.

## Структура папки
- `main.py` — точка входа модуля
- `AGENTS.md` — локальная политика агента
- `README.md` — описание модуля и карта файлов
- `app/` — основная логика модуля
- `ui/` — UI модуля
- `db/` — работа с БД и схема
- `tests/` — тесты
- `prompts/` — промпты, если модуль их использует
- `logs/` — локальные логи, если используются

## Карта файлов
- `app/` — бизнес-логика модуля
- `ui/` — локальный интерфейс
- `db/` — локальная БД и доступ к ней
- `tests/` — локальные тесты
- `logs/` — локальные логи

## Запуск
Команда запуска:
- `python main.py`

Порт:
- 8103

## База данных
У этого модуля своя отдельная БД.

## Основные сценарии
1. Запуск модуля.
2. Работа через локальный UI.
3. Сохранение результатов в собственной БД.
4. Фиксация действий в SQL-реестре задач.

## Ограничения
- модуль не должен брать на себя задачи других модулей;
- README должен обновляться при изменении структуры.
