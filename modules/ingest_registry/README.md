# ingest_registry

## Назначение
Автономный модуль импорта и синхронизации файлов.

Стартовая версия модуля уже умеет:
- запускать свой локальный HTTP UI;
- принимать локальную загрузку файлов;
- хранить реестр файлов в собственной SQLite БД;
- показывать статусы источников `google_drive`, `yandex_disk`, `local_upload`;
- хранить журнал действий модуля.

Стартовая версия модуля пока не умеет:
- выполнять реальный импорт из Google Drive;
- выполнять реальный импорт из Яндекс Диска;
- делать OCR;
- делать классификацию;
- выполнять проектную аналитику.

## Порт
- `8101`

## Запуск
Из корня проекта:

```bash
./modulectl start ingest_registry
```

Проверка статуса и доступности:

```bash
./modulectl status ingest_registry
./modulectl health ingest_registry
```

После запуска UI доступен по адресу:

```text
http://127.0.0.1:8101/
```

## Структура папки
- `main.py` — точка входа, поднимает локальный HTTP сервер модуля.
- `AGENTS.md` — локальная политика агента для `ingest_registry`.
- `README.md` — описание модуля и карта файлов.
- `app/` — app-слой: конфигурация, bootstrap, сервисная логика.
- `db/` — db-слой: схема SQLite и repository для реестра.
- `ui/` — ui-слой: HTTP handler и HTML страницы состояния.
- `data/` — локальные данные модуля, SQLite БД и сохраненные файлы.
- `logs/` — локальные логи запуска и действий.
- `tests/` — директория под будущие тесты модуля.
- `prompts/` — директория под локальные промпты, если понадобятся.

## Карта файлов
- `main.py` — entrypoint.
- `app/config.py` — настройки путей, имени модуля и порта.
- `app/bootstrap.py` — инициализация логирования, БД, сервиса и HTTP сервера.
- `app/service.py` — бизнес-логика модуля: bootstrap, загрузка файла, stub sync, dashboard data.
- `db/schema.py` — SQL схема таблиц `source_status`, `file_registry`, `activity_log`.
- `db/repository.py` — доступ к SQLite БД и операции с реестром.
- `ui/http.py` — обработка `GET /`, `POST /upload`, `POST /sync/<source>`, рендер HTML.
- `data/ingest_registry.sqlite3` — собственная БД модуля.
- `data/storage/bootstrap/welcome.txt` — bootstrap-файл, который подтверждает работу начального реестра.
- `data/storage/uploads/` — каталог локально загруженных файлов.
- `logs/ingest_registry.log` — файл логов модуля.

## UI
Страница состояния показывает:
- общие счетчики по модулю;
- блок локальной загрузки файла;
- блок статусов источников;
- таблицу реестра файлов;
- журнал последних действий.

## База данных
У модуля своя отдельная SQLite БД:

```text
data/ingest_registry.sqlite3
```

Таблицы:
- `source_status` — статусы и время последней синхронизации по источникам.
- `file_registry` — реестр файлов с источником, идентификатором, путем, именем, датой и статусом.
- `activity_log` — журнал ключевых действий модуля.

## Основные сценарии
1. Запустить модуль через `./modulectl start ingest_registry`.
2. Открыть UI на `http://127.0.0.1:8101/`.
3. Загрузить локальный файл через форму upload.
4. Увидеть новую запись в таблице реестра файлов.
5. Проверить статусы источников и выполнить stub sync для каждого источника.

## Что уже подготовлено в стартовой версии
- Есть свой entrypoint через `main.py`.
- Есть разделение на `app`, `db`, `ui`.
- Есть отдельная БД модуля.
- Есть начальный bootstrap-файл в реестре.
- Есть локальная загрузка через UI.
- Есть статусы источников для будущих интеграций Google Drive и Яндекс Диска.

## Ограничения стартовой версии
- Интеграции Google Drive и Яндекс Диска пока представлены только статусами и stub sync.
- Файловый реестр хранит только метаданные и локальный путь.
- Внешние токены и реальные подключения не добавлялись.
