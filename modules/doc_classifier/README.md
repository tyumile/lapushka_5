# doc_classifier

## Назначение
Автономный модуль распознавания и классификации документов.

Модуль:
- берет документы из подготовленного локального источника;
- извлекает текст и базовые признаки;
- определяет тип документа;
- сохраняет отдельно результат распознавания и результат классификации;
- показывает статусы и результаты в собственном UI.

Модуль не делает:
- импорт из внешних источников;
- ведение первичного реестра файлов;
- проектные связи и межмодульную аналитику.

## Структура папки
- `main.py` - точка входа модуля.
- `AGENTS.md` - локальная политика агента для модуля.
- `README.md` - описание модуля и карта файлов.
- `app/` - прикладная логика модуля.
- `db/` - локальная SQLite БД, схема и репозиторий.
- `ui/` - локальный HTTP UI.
- `data/` - каталог для локальной БД модуля.

## Карта файлов
- `main.py` - запускает bootstrap и HTTP UI.
- `app/config.py` - локальная конфигурация модуля, путь к БД и порт.
- `app/bootstrap.py` - старт логирования, инициализация БД, seed и запуск сервера.
- `app/models.py` - модель записи документа.
- `app/pipeline.py` - минимальное распознавание текста и rule-based классификация.
- `app/service.py` - прикладной сервис для dashboard, списка документов и повторного прогона.
- `db/setup.py` - создание таблиц `documents` и `module_events`.
- `db/repository.py` - доступ к SQLite, seed демонстрационных документов и обновление результатов.
- `ui/server.py` - HTTP сервер и маршруты UI.
- `ui/templates.py` - HTML шаблоны страниц.
- `data/doc_classifier.sqlite3` - локальная БД модуля, создается при первом запуске.

## Как запускать
Команда:

```bash
./modulectl start doc_classifier
```

Проверка статуса и доступности:

```bash
./modulectl status doc_classifier
./modulectl health doc_classifier
```

Адрес UI:
- `http://127.0.0.1:8102`

Порт:
- `8102`

## UI
Доступны страницы:
- `/` - страница статуса модуля и агрегированные счетчики.
- `/documents` - список документов со статусами.
- `/documents/<id>` - карточка документа.

На карточке документа показываются:
- исходный статус;
- статус распознавания;
- статус классификации;
- распознанный текст;
- извлеченные признаки;
- результат классификации;
- кнопка повторного прогона документа.

## База данных
У модуля своя локальная SQLite БД:
- путь: `data/doc_classifier.sqlite3`

Основные таблицы:
- `documents` - документы, исходный статус, статус распознавания, статус классификации, распознанный текст и результат классификации.
- `module_events` - локальный журнал событий модуля.

Статусы разделены на три независимых поля:
- `source_status`
- `recognition_status`
- `classification_status`

## Основные сценарии
1. Запустить модуль через `./modulectl start doc_classifier`.
2. Открыть UI на порту `8102`.
3. Посмотреть общий статус модуля на главной странице.
4. Открыть список документов.
5. Перейти в карточку документа и посмотреть результат распознавания и классификации.
6. Повторно прогнать документ без повреждения существующей структуры данных.

## Примечания по стартовой реализации
- Для первого шага используется стандартная библиотека Python без внешних зависимостей.
- Источник документов пока локальный и seeded в БД при первом запуске.
- Классификация выполнена простыми правилами по ключевым словам, чтобы модуль был рабочим и проверяемым.
